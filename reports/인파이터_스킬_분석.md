## 주요 관찰 사항

1. **선행 스킬(preRequiredSkill) 전부 null** — API는 선행 조건을 노출하지 않음. 트리 구조는 `requiredLevel` 기준으로 설계해야 함.
2. **쿨타임은 레벨에 무관하게 고정** — 모든 스킬에서 lv1과 maxlv의 coolTime 동일.
3. **castingTime이 null인 스킬 다수** — API 미제공. 실측값이 있는 스킬만 4개 (분노의 움켜쥠 0.3s, 승천진 0.4s, 섀도우 박서 0.5s, 신의 분노 0.5s). 나머지는 0으로 취급.
4. **데미지 수치는 공격력 대비 %** — value1이 175이면 공격력 × 1.75. 고레벨 스킬일수록 수만 % 단위.

### 선정 기준
- 버프/카운터 제외, 순수 데미지 딜링 스킬만
- 쿨타임 다양성 (2s ~ 45s)
- 트리 깊이 4단계, 총 10개 노드
- castingTime은 API 실측값만 사용 (null → 0)

## 주의: castingTime 데이터 공백

API가 대부분의 스킬에서 `castingTime: null`을 반환하므로, 시뮬레이터의 `base_cast_frames`는 다음 방식 중 선택이 필요합니다.

| 방식 | 장점 | 단점 |
|---|---|---|
| **null → 0 처리** (현재 제안) | API 데이터만 사용, 정확한 실측 불필요 | 시전시간이 0이면 동일 쿨타임 스킬 간 우선순위가 데미지로만 결정됨 |
| 게임 내 직접 측정 | 현실에 가까운 시뮬레이션 | 28개 스킬 모두 수동 측정 필요 |
| 임의 추정값 부여 | 시전감 재현 | 근거 없는 수치로 결과 왜곡 가능 |

**M1 샘플에서는 null → 0 처리를 기본으로 사용**하고, 이후 필요 시 게임 내 측정값으로 교체하는 방향을 권장합니다.
