# M0 보고서 — 환경 세팅

## 목표

세 레이어(C 코어 / Node.js 서버 / React 프론트엔드)가 각각 독립적으로 실행되고, 서버가 C 실행파일을 subprocess로 호출해 응답을 반환하는 것을 확인한다.

---

## 완료된 작업

### 1. 디렉토리 구조 생성

```
workspace/
├── core/       C 코어 소스 및 빌드 산출물
├── server/     Node.js Express 서버
├── frontend/   Vite + React 프론트엔드
└── reports/    마일스톤별 보고서 (이 파일)
```

### 2. C 코어 빌드 환경

- **컴파일러**: `gcc 15.2.0` (MinGW-W64)
- **빌드 도구**: `mingw32-make` — Windows 환경에서 GNU Make 대체. Makefile 문법과 동작은 `make`와 동일.
- **Makefile 구성**:
  - 컴파일 플래그: `-std=c99 -Wall -Wextra -pedantic`
  - 소스: `main.c`, `cJSON.c` → `simulator.exe`
- **첫 번째 `main.c`**: stdin 입력 없이 `{"status":"alive"}` 를 stdout으로 출력하는 최소 골격. M1~M4에서 점진적으로 교체된다.

### 3. cJSON 라이브러리

- **출처**: [github.com/DaveGamble/cJSON](https://github.com/DaveGamble/cJSON) 공식 릴리즈
- **배치**: `core/cJSON.h`, `core/cJSON.c`
- **규칙**: 이 두 파일은 이후 절대 수정하지 않는다. 래퍼 함수는 별도 `json_parser.c`에 작성한다.
- **이유**: cJSON은 단일 헤더+소스 구조라 외부 빌드 시스템 없이 프로젝트에 직접 포함 가능하다.

### 4. Node.js Express 서버

- **파일**: `server/app.js`, `server/package.json`
- **패키지**: `express ^4.18`, `cors ^2.8`
- **`GET /ping` 라우트 동작 흐름**:
  1. `child_process.spawn`으로 `simulator.exe` 실행
  2. 5초 타임아웃 타이머 시작
  3. `child.stdin.end()` — 현재 main.c는 stdin 없이 바로 출력
  4. stdout에서 JSON 수신 후 `{"status":"ok","c_core":"alive"}` 반환
- **에러 처리**:
  - 실행파일 없음 → 503
  - 5초 초과 → 504
  - exit code ≠ 0 → 500

### 5. Vite + React 프론트엔드

- **scaffold**: `npm create vite@latest frontend -- --template react`
- **주의사항**: `@eslint/js` 등 ESLint 10.x 패키지가 Node 23을 지원 목록에 명시하지 않아 `warn EBADENGINE` 경고가 출력됨. Vite와 React 자체에는 영향 없으며, dev server 기동 정상 확인.

## 완료 기준 검증

```
$ curl http://localhost:5000/ping
{"status":"ok","c_core":"alive"}          ← 통과

$ curl http://localhost:5173
HTTP 200                                   ← 통과
```

---

## 아키텍처 흐름 (현재 시점)

```
브라우저 (localhost:5173)
    ↓  (M6~M7에서 연동 예정)
Node.js (localhost:5000)
    ↓  GET /ping → spawn simulator.exe
C 실행파일 → stdout: {"status":"alive"}
    ↑
Node.js → {"status":"ok","c_core":"alive"}
```

현재 C 코어는 아직 최소 골격 상태이며, M1~M4를 거쳐 실제 트리+시뮬레이터로 교체된다.
